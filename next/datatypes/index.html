Contextual Data Typesâ€‹

  • ZEnvironment â€” ZEnvironment[R] is a built-in type-level map for the ZIO data type which is responsible for maintaining the environment of a ZIO effect.
  • ZLayer â€” ZLayer[-RIn, +E, +ROut] is a recipe to build an environment of type ROut, starting from a value RIn, and possibly producing an error E during creation.
    • RLayer â€” RLayer[-RIn, +ROut] is a type alias for ZLayer[RIn, Throwable, ROut], which represents a layer that requires RIn as its input, it may fail with Throwable value, or returns ROut as its output.
    • ULayer â€” ULayer[+ROut] is a type alias for ZLayer[Any, Nothing, ROut], which represents a layer that doesn't require any services as its input, it can't fail, and returns ROut as its output.
    • Layer â€” Layer[+E, +ROut] is a type alias for ZLayer[Any, E, ROut], which represents a layer that doesn't require any services, it may fail with an error type of E, and returns ROut as its output.
    • URLayer â€” URLayer[-RIn, +ROut] is a type alias for ZLayer[RIn, Nothing, ROut], which represents a layer that requires RIn as its input, it can't fail, and returns ROut as its output.
    • TaskLayer â€” TaskLayer[+ROut] is a type alias for ZLayer[Any, Throwable, ROut], which represents a layer that doesn't require any services as its input, it may fail with Throwable value, and returns ROut as its output.

Concurrencyâ€‹

Fiber Primitivesâ€‹

  • Fiber â€” A fiber value models an IO value that has started running, and is the moral equivalent of a green thread.
  • FiberRef â€” FiberRef[A] models a mutable reference to a value of type A. As opposed to Ref[A], a value is bound to an executing Fiber only. You can think of it as Java's ThreadLocal on steroids.
  • Fiber.Status â€” Fiber.Status describe the current status of a Fiber.
  • FiberId â€” FiberId describe the unique identity of a Fiber.

Concurrency Primitivesâ€‹

  • Hub â€” A Hub is an asynchronous message hub that allows publishers to efficiently broadcast values to many subscribers.
  • Promise â€” A Promise is a model of a variable that may be set a single time, and awaited on by many fibers.
  • Semaphore â€” A Semaphore is an asynchronous (non-blocking) semaphore that plays well with ZIO's interruption.
  • Ref â€” Ref[A] models a mutable reference to a value of type A. The two basic operations are set, which fills the Ref with a new value, and get, which retrieves its current content. All operations on a Ref are atomic and thread-safe, providing a reliable foundation for synchronizing concurrent programs.
  • Ref.Synchronized â€” Ref.Synchronized[A] models a mutable reference to a value of type A in which we can store immutable data, and update it atomically and effectfully.
  • Queue â€” A Queue is an asynchronous queue that never blocks, which is safe for multiple concurrent producers and consumers.

Synchronization aidsâ€‹

  • ConcurrentMap â€” A Map wrapper over java.util.concurrent.ConcurrentHashMap
  • ConcurrentSet â€” A Set implementation over java.util.concurrent.ConcurrentHashMap
  • CountdownLatch â€” A synchronization aid that allows one or more fibers to wait until a set of operations being performed in other fibers completes.
  • CyclicBarrier â€” A synchronization aid that allows a set of fibers to all wait for each other to reach a common barrier point.

STMâ€‹

  • STM â€” An STM represents an effect that can be performed transactionally resulting in a failure or success.
  • TArray â€” A TArray is an array of mutable references that can participate in transactions.
  • TSet â€” A TSet is a mutable set that can participate in transactions.
  • TMap â€” A TMap is a mutable map that can participate in transactions.
  • TRef â€” A TRef is a mutable reference to an immutable value that can participate in transactions.
  • TPriorityQueue â€” A TPriorityQueue is a mutable priority queue that can participate in transactions.
  • TPromise â€” A TPromise is a mutable reference that can be set exactly once and can participate in transactions.
  • TQueue â€” A TQueue is a mutable queue that can participate in transactions.
  • TReentrantLock â€” A TReentrantLock is a reentrant read / write lock that can be composed.
  • TSemaphore â€” A TSemaphore is a semaphore that can participate in transactions.

Resource Managementâ€‹

  • Scope â€” A scope in which resources can safely be used.
  • ZPool â€” An asynchronous and concurrent generalized pool of reusable resources.

Streamingâ€‹

  • ZStream â€” ZStream is a lazy, concurrent, asynchronous source of values.
    • Stream â€” Stream[E, A] is a type alias for ZStream[Any, E, A], which represents a ZIO stream that does not require any services, and may fail with an E, or produce elements with an A.
  • ZSink â€” ZSink is a consumer of values from a ZStream, which may produce a value when it has consumed enough.
    • Sink â€” Sink[InErr, A, OutErr, L, B] is a type alias for ZSink[Any, InErr, A, OutErr, L, B].
  • ZPipeline â€” ZPipeline is a polymorphic stream transformer.
  • SubscriptionRef â€” SubscriptionRef[A] contains a current value of type A and a stream that can be consumed to observe all changes to that value.

Miscellaneousâ€‹

  • Chunk â€” Chunk is a fast, pure alternative to Arrays.
  • Schedule â€” Schedule is a model of a recurring schedule, which can be used for repeating successful IO values, or retrying failed IO values.
  • Supervisor â€” Supervisor[A] is allowed to supervise the launching and termination of fibers, producing some visible value of type A from the supervision.
